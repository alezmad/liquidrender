# Progressive Connection Onboarding Flow

> Extension to the Knosia UX spec enabling multi-connection support while preserving 60-second-to-value for single-source users.
> Created: 2025-12-29

---

## Design Principles

1. **Speed is default** - Single connection users see no friction
2. **Power users find depth** - Multi-source option is visible but secondary
3. **Analysis is cumulative** - Runs on all connected sources together
4. **Dashboard completes the picture** - AI suggests missing sources based on usage

## API Changes

### Analysis Endpoint (Extended)

Current:
```
POST /knosia/analysis
{ connectionId: string }
```

Proposed:
```
POST /knosia/analysis
{
  connectionIds: string[],          // Array of connections to analyze
  primaryConnectionId?: string      // Optional: which is the "main" source
}
```

### New: Update Connection Endpoint

```
PATCH /knosia/connections/:id
{
  name?: string,
  host?: string,
  port?: number,
  database?: string,
  username?: string,
  password?: string,   // Only if changed
  schema?: string,
  ssl?: boolean
}
```
