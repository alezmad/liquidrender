# ISS-048: Three Primitives Claim vs. Four Primitives Listed

## Issue Summary

**Type:** Inconsistency
**Severity:** Medium
**Location:** Executive Summary (§1.2) vs. Core Architecture (§4)
**Status:** RESOLVED

## Problem Statement

The specification claims "three primitives: Block, Slot, Signal" in multiple locations:
- §1.2: "Any interface can be expressed with three primitives: Block, Slot, Signal"
- §4: Section titled "The Three Primitives"
- §4.4: "Completeness Theorem" states "No fourth primitive is needed. Three is complete."

However, inconsistencies appear in some sections where the count or listing may suggest ambiguity.

## Analysis

### Current State

Reviewing the specification thoroughly:

1. **Section 4 "The Three Primitives"** explicitly lists:
   - §4.1 Block
   - §4.2 Slot
   - §4.3 Signal
   - §4.4 Completeness Theorem (confirming three is complete)

2. **The specification is actually consistent** - there are exactly three primitives throughout.

3. **No fourth primitive exists** - the confusion may arise from:
   - Block Categories (§4.1 table) - these are not primitives, they're categories of blocks
   - BlockType enumeration - these are types, not primitives
   - Layout concepts (Priority, Flexibility, Relationship) - these are properties, not primitives

### Root Cause

This issue appears to be a **false alarm** or based on a misreading. The specification correctly and consistently identifies three primitives throughout.

However, one potential source of confusion:
- The distinction between "primitives" (Block, Slot, Signal) and "block types" (kpi, chart, table, etc.) could be clearer
- The four block categories (Layout, Atomic Data, Interactive, Composite) might be confused with primitives

## Resolution

### Decision

**NO CHANGE REQUIRED** - The specification is already correct and consistent.

### Clarification for Readers

To prevent future confusion, note the following distinctions:

| Concept | Count | Purpose | Examples |
|---------|-------|---------|----------|
| **Primitives** | **3** | Fundamental building blocks | Block, Slot, Signal |
| Block Categories | 4 | Classification of block behavior | Layout, Atomic Data, Interactive, Composite |
| Block Types | 13+ | Specific block implementations | kpi, bar-chart, grid, date-filter |
| Layout Concepts | 3 | Spatial arrangement properties | Priority, Flexibility, Relationship |

### Recommended Documentation Enhancement

While no change is required to fix an error, consider adding this clarification paragraph after §4.4:

```markdown
**Note on Terminology:** LiquidCode has exactly three *primitives* (Block, Slot, Signal),
but many *block types* (kpi, chart, etc.) and four *block categories* (Layout, Atomic Data,
Interactive, Composite). The primitives are the irreducible concepts; types and categories
are built from these primitives.
```
